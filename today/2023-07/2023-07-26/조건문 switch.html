switch 문

값에 따라 서로 다른 코드를 실행할 때, if-else 문보다 switch문이 적합하다. 

        var fruits = "사과"; 

        var price = 0; 


        switch(fruits){ 

            case "사과": 

                price=100; 

                break; 

            case "바나나": 

                price=200; 

                break; 

            case "포도": 

                price=300; 

                break; 

            default: 

                document.write("팔지않습니다.") 

                price=0; 

        } 

        

        document.write(fruits + "는" + price + "원 입니다.")


case 문의 값

case 문의 값은 const로 선언된 상수나, 리터럴만 가능하다.

case 1 :
case 2.7 :
case "asdf" :
case true :
case 2+3 :

하지만 case 문의 값에 변수나 식은 사용할 수 없다. 다음은 잘못된 사례이다.

case a : ----- //변수로 선언된 a 는 사용불가
case a > 3 : ----- // a > 3 은 식이므로 사용불가


switch 문에서 break 의 역할

case 문 내의 실행 문장을 실행한 후 break문을 만나지 못하면 만날때까지 그 아래 다른 case의 실행 문장들을 계속 실행하여 내려간다.
